### Database Schema - Firestore

```yaml
users/{userId}
  ├── email: string
  ├── name: string
  ├── createdAt: timestamp
  └── preferences: object

expenses/{expenseId}
  ├── userId: string (indexed)
  ├── title: string
  ├── amount: number
  ├── category: string
  ├── date: timestamp (indexed)
  ├── note: string (optional)
  ├── receiptUrl: string (optional)
  ├── isRecurring: boolean
  └── createdAt: timestamp

budgets/{budgetId}
  ├── userId: string (indexed)
  ├── monthlyLimit: number
  ├── categoryLimits: map
  │   └── category: number
  ├── month: number
  ├── year: number
  └── alerts: array

recurring_expenses/{recurringId}
  ├── userId: string (indexed)
  ├── title: string
  ├── amount: number
  ├── category: string
  ├── frequency: string (MONTHLY, WEEKLY, DAILY)
  ├── nextDueDate: timestamp
  ├── isActive: boolean
  └── reminderDaysBefore: number

income/{incomeId}
  ├── userId: string (indexed)
  ├── source: string
  ├── amount: number
  ├── date: timestamp (indexed)
  ├── note: string (optional)
  └── isRecurring: boolean
```

### Security Rules - Firestore

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // User data - only own data
    match /users/{userId} {
      allow read, write: if request.auth != null && 
                            request.auth.uid == userId;
    }
    
    // Expenses - only own expenses
    match /expenses/{expenseId} {
      allow read, write: if request.auth != null && 
                            request.auth.uid == resource.data.userId;
      allow create: if request.auth != null && 
                       request.auth.uid == request.resource.data.userId;
    }
    
    // Similar rules for budgets, income, recurring_expenses
  }
}
```

## 🚀 Installation & Setup

### Prerequisites

- **Flutter SDK** 3.10.7 or higher ([Download](https://flutter.dev/docs/get-started/install))
- **Dart SDK** 3.0+ (included with Flutter)
- **Android Studio** or **VS Code** with Flutter extension
- **Firebase Account** (free tier available at [console.firebase.google.com](https://console.firebase.google.com))
- **Android Device/Emulator** or **iOS Device/Simulator**

### Step 1️⃣: Clone the Repository

```bash
git clone https://github.com/lucifers-0666/KHARCHA-Expense-Tracker_APP.git
cd KHARCHA-Expense-Tracker_APP
```

### Step 2️⃣: Install Flutter Dependencies

```bash
flutter pub get
```

### Step 3️⃣: Firebase Configuration

1. **Create Firebase Project:**
   - Go to [Firebase Console](https://console.firebase.google.com)
   - Click "Create New Project" → Name it "KHARCHA"
   - Enable Google Analytics (optional)

2. **Enable Authentication:**
   - Navigate to **Authentication** → **Sign-in Method**
   - Enable **Email/Password** provider
   - Optionally enable **Google Sign-In**

3. **Create Firestore Database:**
   - Go to **Firestore Database** → **Create Database**
   - Select region (closest to your location)
   - Start in **Test Mode** (we'll secure later)

4. **Download Configuration Files:**

   **For Android:**
   - Go to Project Settings → General
   - Download `google-services.json`
   - Place in `android/app/`

   **For iOS:**
   - Go to Project Settings → General
   - Download `GoogleService-Info.plist`
   - Place in `ios/Runner/` (add via Xcode)

5. **Configure Firebase CLI:**
   ```bash
   flutter pub add firebase_core firebase_auth cloud_firestore
   flutterfire configure
   ```

### Step 4️⃣: Update Firestore Security Rules

1. Go to **Firestore Database** → **Rules**
2. Replace with:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /expenses/{expenseId} {
      allow read, write: if request.auth != null && 
                            request.auth.uid == resource.data.userId;
    }
    match /budgets/{budgetId} {
      allow read, write: if request.auth != null && 
                            request.auth.uid == resource.data.userId;
    }
    match /recurring_expenses/{recurringId} {
      allow read, write: if request.auth != null && 
                            request.auth.uid == resource.data.userId;
    }
    match /income/{incomeId} {
      allow read, write: if request.auth != null && 
                            request.auth.uid == resource.data.userId;
    }
    match /users/{userId} {
      allow read, write: if request.auth != null && 
                            request.auth.uid == userId;
    }
  }
}
```

### Step 5️⃣: Run the App

```bash
# Run on Android device/emulator
flutter run

# Or run on iOS device/simulator
flutter run -d iPhone
```

### Step 6️⃣ (Optional): Build for Release

```bash
# Android APK
flutter build apk --release

# Android App Bundle (for Play Store)
flutter build appbundle --release

# iOS IPA
flutter build ipa --release
```

Generated files location:
- Android APK: `build/app/outputs/flutter-apk/app-release.apk`
- App Bundle: `build/app/outputs/bundle/release/app-release.aab`

## 📱 Complete Usage Guide

### 🎯 Adding Your First Expense

1. Tap the **floating "+" button** on home screen
2. Enter **amount** (e.g., ₹450 for lunch)
3. Select **category** (Food, Transport, Bills, etc.)
4. Add optional **note** ("Lunch with team")
5. Choose **date** (defaults to today)
6. Tap **Save** → Entry appears instantly
7. See updated dashboard

**💡 Pro Tip:** Expenses sync automatically if online. Work offline anytime!

### 💳 Setting Up Your Budget

1. Navigate to **Budget** tab
2. Tap **"Set Monthly Budget"**
3. Enter total monthly limit (e.g., ₹25,000)
4. **Allocate by category:**
   - Food: ₹7,000
   - Transport: ₹3,000
   - Bills: ₹10,000
   - Shopping: ₹3,000
   - Entertainment: ₹2,000
5. Tap **Save**
6. Get smart alerts at 75%, 90%, 100% threshold

**💡 Pro Tip:** Review category spending monthly. Adjust limits based on trends!

### 📊 Viewing Your Analytics

1. Open **Analytics** tab
2. View **spending overview** for the month
3. Explore **category breakdown** pie chart
4. Check **6-month trends** line graph
5. Review **top expenses** and **financial health score**
6. Drill down into any category for details

**💡 Pro Tip:** Compare month-over-month trends to identify savings opportunities!

### 🔄 Setting Up Recurring Expenses

1. Go to **Recurring** section
2. Tap **Add Recurring Expense**
3. Fill in:
   - Title: "Netflix Subscription"
   - Amount: ₹649
   - Frequency: Monthly
   - Next Due Date: [Select]
4. Enable **notifications** (2 days before)
5. Save

**💡 Pro Tip:** Add all recurring expenses at once. Never miss a bill!

### 📄 Exporting Your Reports

1. Go to **Settings** → **Reports & Export**
2. Choose **date range**
3. Select export format:
   - 📑 **PDF** - Professional reports with charts
   - 📊 **CSV** - For Excel analysis
4. Tap **Export**
5. Share via WhatsApp, Email, or save locally

**💡 Pro Tip:** Use PDF for tax filing, CSV for detailed analysis!

## 🐛 Troubleshooting & FAQ

### Q: App not syncing with Firebase?
**A:** 
- Check internet connection
- Verify Firestore is enabled (Firebase Console)
- Check quota limits haven't exceeded
- Review Firestore security rules
- Clear app cache: Settings → Apps → KHARCHA → Clear Cache

### Q: "NoSuchMethodError" when opening app?
**A:** 
- Run `flutter clean`
- Run `flutter pub get`
- Rebuild: `flutter run`
- If persists, check Firebase configuration

### Q: Charts not displaying?
**A:** 
- Ensure you have at least 2 expenses on different dates
- Check data is properly synced from Firestore
- Verify chart data permissions

### Q: How do I backup my data?
**A:** 
- Settings → Backup & Export → Choose PDF/CSV
- App automatically syncs to Firebase Cloud
- Use Settings → Account → Export All Data

### Q: Can I use offline without account?
**A:** 
- Limited mode: Local-only tracking
- Recommended: Create Firebase account for cloud sync & multi-device access

### Q: How to transfer data to new device?
**A:** 
- Settings → Backup & Export → Save PDF/CSV
- On new device: Login with same account → Auto-sync begins
- Or import CSV from previous device

### Q: Why is the app size so large?
**A:** 
- Flutter framework: ~4 MB
- Firebase SDK: ~8 MB
- Charts & UI libraries: ~3 MB
- **Total:** ~15 MB (optimized for features)
